# 激光铺面层指南

目标：节奏点继续在 ArrowVortex 里编辑和预览，Godot 只根据额外的 NOTES layer 解释激光规则。

## Layer 命名

在同一个 `.sm` 文件里保留原来的主谱面，再新增两个可选 NOTES layer：

- `LaserEcho`：先按谱面节奏播预警，下一小节按同样位置和节奏发射。
- `LaserChord`：同一行的多个位置一起播预警，下一小节按位置号发射。

解析器会在 NOTES 的 metadata 里查找 `LaserEcho` / `LaserChord`，大小写、空格、下划线和短横线都不敏感。推荐把 NOTES 的 description 写成 `LaserEcho` 或 `LaserChord`。

## 位置含义

激光 layer 的 4 列不再表示原来的 LDUR 动作，而是表示激光位置：

```text
第1列 = 1号位
第2列 = 2号位
第3列 = 3号位
第4列 = 4号位
```

普通主谱面仍然保持现有映射：左/下为 HIT，上为 GUARD，右为 DODGE。

## 动画和音效

激光现在拆成两段：

- 预警段：播放 `guard_warn_scene`，也就是 `warn.tscn`。
- 攻击段：播放 `guard_scene`，也就是 `laser.tscn`。

音效同步拆分：

- 播 `warn.tscn` 时播放激光预警音效。
- 播 `laser.tscn` 时播放激光攻击音效。

常规 GUARD 模式下，第 1 拍播 `warn.tscn`，第 2 拍也就是判定拍播 `laser.tscn`。判定仍然挂在第 2 拍，不会因为预警提前而改变。

`laser.tscn` 会提前开播，让攻击动画的第 3 帧，也就是 frame index `2`，正好和判定拍/节奏重拍重合。

## LaserEcho

示例：2 号位按四分拍预警 4 次，下一小节按同样节奏发射。

```text
0100
0000
0100
0000
0100
0000
0100
0000
,
```

这一层里的拍点只播 `warn.tscn`。解析器会在下一小节展开同样节奏的 fire note，这些 fire note 只播 `laser.tscn`，不会额外提前一拍再播 warn。

如果要半拍、三连音或更密的节奏，直接在 ArrowVortex 里切换网格。解析器按小节行数计算拍点，不需要手写 beat 数字。

## LaserChord

示例：1、3、4 号位同时预警，下一小节第 1、3、4 拍分别发射。

```text
1011
0000
0000
0000
,
```

这一行的多个位置会同时播 `warn.tscn`。之后位置号会映射到下一小节内的发射拍：

```text
1号位 -> 第1拍播 laser.tscn
2号位 -> 第2拍播 laser.tscn
3号位 -> 第3拍播 laser.tscn
4号位 -> 第4拍播 laser.tscn
```

## 场景配置

旧谱的普通 GUARD 激光仍使用 `guard_position_nodes`，不会因为新功能改变轮换位置。

常规 GUARD 的预警位置单独使用 `guard_warn_position_nodes`。主场景里当前映射是：

```text
Laser1 -> LaserWarn3
Laser2 -> LaserWarn4
```

新激光 layer 使用 `laser_pattern_position_nodes` 和 `laser_pattern_warn_position_nodes`。预警和发射默认可以指向同一组 4 个 Laser 节点；以后如果换布局，只需要调整这两个数组。
