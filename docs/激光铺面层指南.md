# 激光铺面指南

目标：所有防御谱面都写在 `.sm` 的 `Main` 谱面里。旧的 `LaserEcho` / `LaserChord` 额外 NOTES layer 已弃用，运行时会忽略。

## Main 列含义

Main 谱面仍使用 4 列：

```text
第 1 列 = 导弹 / 激光位置 1
第 2 列 = 导弹 / 激光位置 2
第 3 列 = 激光位置 3
第 4 列 = 蓄力 / 激光位置 4
```

单行只有第 1 或第 2 列有音符时，仍按导弹 HIT 解析。

单行只有第 4 列有音符时，仍按蓄力 DODGE 解析。

单行只有第 3 列有音符时，按激光 Echo 规则解析。

单行有多个 active columns 时，整行按激光 Chord 规则解析，不再生成导弹或蓄力 note。

## Echo 规则

Main 中单个第 3 列激光音符表示激光发射时间。解析器会自动提前 2 拍生成预警：

```text
0010
```

等价于：

```text
warning = fire - 2.0 beats
fire = authored beat
```

## Chord 规则

Main 中同一行多个 active columns 表示一次 Chord 预警。预警在该行时间同时出现，随后按位置递进发射：

```text
1111
```

等价于：

```text
位置 1 -> warning + 2.0 beats 发射
位置 2 -> warning + 2.5 beats 发射
位置 3 -> warning + 3.0 beats 发射
位置 4 -> warning + 3.5 beats 发射
```

示例：

```text
1111
0000
0000
0000
,
```

这一行会同时显示 4 个预警，然后按半拍间隔依次发射。

## 旧 Layer

`LaserEcho` 和 `LaserChord` NOTES layer 仍可留在旧文件里，但运行时不会读取。迁移旧谱时，把旧 layer 中的激光音符移到 Main：

- 单发 Echo：移成 Main 的单个第 3 列音符 `0010`。
- Chord：移成 Main 的同一行多列音符，例如 `1111` 或 `1101`。

## 场景配置

Main 里的新激光仍使用：

- `laser_pattern_warn_position_nodes`
- `laser_pattern_position_nodes`

普通导弹和蓄力仍使用原来的 HIT / DODGE 运行时路径。
