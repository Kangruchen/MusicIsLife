# music-is-life 开发文档

> 最后更新：2026-03-02
> 引擎版本：Godot 4.x（GDScript 静态类型）

---

## 一、项目概述

**music-is-life** 是一款节奏类 RPG 游戏。玩家跟随音乐节拍进行防御/攻击操作，并在 BOSS 精力耗尽时进入专属攻击阶段对 BOSS 发动攻击。游戏融合了音游判定、角色血量管理、多轨道音乐、和蓄力战斗系统。

## 三、架构规则

### 3.1 EventBus 信号总线

**所有跨子树或多个监听者的通信必须通过 `EventBus`**，禁止跨层直接调用节点方法。

同级兄弟节点（同属 `GameManager`）之间可用直接引用（`@onready`），但仍优先通过 EventBus 通信。

#### 已注册信号一览

| 分类 | 信号 | 参数 | 说明 |
|------|------|------|------|
| 音乐/节拍 | `music_started` | — | 音乐开始播放 |
| | `beat_hit` | `beat_number: float, note: Variant` | 每个节拍触发 |
| | `chart_loaded` | `chart: Variant` | 铺面加载完成 |
| 输入/判定 | `judgment_made` | `track: int, judgment: int, timing_diff: float` | 判定结果 |
| | `defense_key_pressed` | `track: int` | 防御阶段按键 |
| | `miss_triggered` | `track: int` | MISS 触发 |
| 攻击阶段 | `attack_performed` | `attack_type: int` | 执行攻击 |
| | `attack_phase_started` | — | 攻击阶段开始 |
| | `attack_phase_ended` | — | 攻击阶段结束 |
| 血量/状态 | `player_health_updated` | `current, maximum` | 玩家血量变化 |
| | `boss_health_updated` | `current, maximum` | BOSS 血量变化 |
| | `boss_energy_updated` | `current, maximum` | BOSS 精力变化 |
| | `boss_energy_depleted` | — | BOSS 精力归零 |
| | `player_died` | — | 玩家死亡 |
| | `boss_defeated` | — | BOSS 被击败 |
| UI 指令 | `show_attack_ui_requested` | — | 显示攻击 UI |
| | `hide_attack_ui_requested` | — | 隐藏攻击 UI |
| | `show_beat_track_requested` | — | 显示节拍轨道 |
| | `spawn_beat_note_requested` | `interval, target_time` | 生成节拍音符 |
| | `show_return_countdown_requested` | `count: int` | 显示返回倒计时 |
| | `show_pause_countdown_requested` | `bi: float` | 显示暂停倒计时 |
| | `play_beat_flash_requested` | `bi, beat_count` | 播放节拍闪光 |
| | `hide_pause_effects_requested` | — | 隐藏暂停特效 |

#### 共享状态

`EventBus.beat_interval: float` — 由 `BeatManager` 在音乐开始后写入，全局可读。

### 3.2 资源（Resource）驱动配置

游戏的所有数值配置均使用 Godot `Resource` 资源文件（`.tres`），在 Inspector 中填写，不硬编码在脚本中：

| 资源类 | 配置文件 | 用途 |
|--------|---------|------|
| `MusicConfig` | — | 多轨道音乐路径与音量 |
| `AttackConfig` | `config/attack_config.tres` | 攻击阶段各攻击数值 |
| `CharacterAnimConfig` | `config/character_anim_config.tres` | 角色动画名称映射 |
| `TrackAnimationConfig` | `config/track_animation_config.tres` | 轨道特效场景与动画名 |
| `KeySoundConfig` | `config/key_sound_config.tres` | 按键音效与音量 |
| `Chart` | `assets/charts/example_chart.tres` | 铺面数据（可通过 SM 文件生成） |

### 3.3 GDScript 编码规范

- **类型提示**：所有变量、参数、返回值必须显式声明类型。
- **命名**：变量/函数用 `snake_case`，类/节点用 `PascalCase`，常量用 `UPPER_SNAKE_CASE`。
- **节点获取**：统一在 `_ready()` 中用 `@onready` 获取，禁止在 `_process` 中重复调用 `get_node()`。
- **节点间通信**：优先使用 EventBus 信号解耦，避免跨层节点直接调用。
- **可复用元素**：使用 `PackedScene` 实例化（如 `NoteVisual.tscn`、`bling.tscn`、`warn.tscn`）。
- **常驻 UI 元素**：在场景树中直接创建对应节点，而非代码动态生成。

## 四、核心系统详解

### 4.1 音符与铺面系统

#### 数据结构

- **`Note`**（Resource）：单个音符，包含 `beat_time`（秒）、`beat_number`（拍号）、`type`（NoteType）。
- **`Chart`**（Resource）：铺面，包含 `chart_name`、`bpm`、`offset`、`notes: Array[Note]`。

#### 音符类型（`Note.NoteType`）

| 枚举 | 按键 | 轨道 | Bling 颜色 |
|------|------|------|-----------|
| `GUARD` | J | 第一轨道 | 蓝色 |
| `HIT` | I | 第二轨道 | 红色 |
| `DODGE` | L | 第三轨道 | 绿色 |

#### 铺面加载优先级（BeatManager）

1. 若 `chart_sm_path` 非空 → `SMFileLoader.load_from_sm()` 加载 `.sm` 文件
2. 若 `generate_test_chart = true` → 随机生成测试铺面
3. 加载后通过 `EventBus.chart_loaded` 广播

#### .sm 文件支持

`SMFileLoader` 支持 StepMania `.sm` 格式，解析 `TITLE`、`ARTIST`、`BPMS`（仅首个 BPM，不支持变速）、`OFFSET`、音符数据。SM 的 offset 取绝对值使用。

铺面文件路径：`assets/charts/AISample.sm`、`assets/charts/demo2.sm`

### 4.2 节拍检测（BeatManager）

- 依据 `bpm` 和 `offset` 计算 `beat_interval`（秒）。
- `_process()` 中轮询 `music_player.get_playback_position()` + `AudioServer.get_time_to_next_mix()` 检测节拍。
- 每次节拍触发 `EventBus.beat_hit(beat_number, note)`。
- 支持暂停/恢复（`is_paused`、`pause_start_time`、`total_pause_duration`）。

#### 用户延迟校准

- 通过 `OffsetCalibration.tscn` 校准，结果保存到 `user://settings.cfg`（`audio/offset`，单位ms）。
- `BeatManager._ready()` 时加载并叠加到铺面 offset，所有音符 `beat_time` 同步偏移。
- 快捷键 **O** 从主场景跳转到校准场景。

### 4.3 判定系统（InputManager）

#### 判定时间窗口

| 等级 | 范围 | GameConstants 常量 |
|------|------|-------------------|
| PERFECT | < 50 ms | `PERFECT_WINDOW = 0.050` |
| GREAT | 50–100 ms | `GREAT_WINDOW = 0.100` |
| GOOD | 100–150 ms | `GOOD_WINDOW = 0.150` |
| MISS | > 200 ms 未击中 | `MISS_THRESHOLD = 0.200` |

#### 游戏阶段状态机（`PhaseState`）

```
DEFENSE（防御阶段，正常游戏）
    │  boss 精力耗尽
    ▼
PAUSED（准备阶段，暂停 4 拍倒计时）
    │  倒计时结束
    ▼
ATTACK（攻击阶段，16 拍输入）
    │  攻击阶段结束
    ▼
DEFENSE（回到防御阶段）
```

按键映射：`note_guard`(J) → GUARD，`note_hit`(I) → HIT，`note_dodge`(L) → DODGE。

### 4.4 攻击阶段系统

#### 时间结构（共 24 拍，按 `GameConstants`）

| 阶段 | 拍数 | 常量 |
|------|------|------|
| 准备倒计时 | 4 拍 | `COUNTDOWN_BEATS = 4` |
| 攻击输入 | 16 拍 | `INPUT_BEATS = 16` |
| 退出倒计时 | 4 拍 | `EXIT_BEATS = 4` |
| **总计** | **24 拍** | `TOTAL_ATTACK_BEATS = 24` |

Drum 轨道第 9 小节（第 32 拍）起启用：`DRUM_START_BEAT = 32`。

#### 攻击类型（`AttackType`）

| 枚举 | 按键 | 效果 |
|------|------|------|
| `LIGHT` | J（GUARD 轨道） | 轻攻击，消耗少伤害适中 |
| `HEAVY` | I（HIT 轨道） | 重攻击，消耗 2 拍，高伤害 |
| `HEAL` | L（DODGE 轨道） | 回复玩家血量 |
| `ENHANCE` | 不输入/错拍 | 蓄力，下次攻击升级为蓄力版本 |

蓄力版本（`is_next_attack_charged = true`）：消耗更少，伤害更高（见 `AttackConfig`）。

#### 节拍驱动方式

攻击阶段节拍由绝对时间数组 `_attack_beat_abs_times` 驱动（`_process` 中比对），而非 Timer，避免漂移。

### 4.5 多轨道音乐播放（MusicPlayer）

- 同步播放三条轨道：`main_player`（主旋律）、`drum_player`（鼓点）、`bass_player`（贝斯）。
- 配置来自 `MusicConfig` 资源，各轨道可独立开关并设置音量（dB）。
- 攻击阶段替换 drum 轨道为专用文件（`AISample_drum.mp3`），阶段结束后恢复。
- MISS 特效：LowPass 滤波（500 Hz）+ 音量降至 -10.46 dB，持续 0.3 秒后恢复（`enable_miss_audio_effect` 开关控制）。
- `get_playback_position()` 用于 BeatManager 时间同步。

### 4.6 轨道视觉系统（TrackManager）

#### 音符生成提前量（拍数）

| 轨道 | 提前量 | 节奏 |
|------|-------|------|
| GUARD（J） | 2 拍 | 静止1拍 + 移动1拍 |
| HIT（I） | 3 拍 | 静止1拍 + 移动2拍 |
| DODGE（L） | 4 拍 | 静止1拍 + 移动3拍 |

- 生成 X 坐标：视口宽度 + 100px（动态计算）
- Bling 特效按行排列：蓝色第一行（J），红色第二行（I），绿色第三行（L）

#### 自定义轨道动画（TrackAnimationConfig）

每种轨道可配置：
- `warn_scene`（预警特效，提前 1 拍）
- `scene`（主动画场景，需含 `AnimatedSprite2D` 子节点）
- `animation_name`
- `attack_end_frame`（对齐判定时刻的关键帧，-1 表示不对齐）

未配置则使用默认 Bling 特效（`bling.tscn`）。

音符视觉 `note_visual_enabled = false`（当前暂停使用传统滑动音符）。

### 4.7 角色动画（Character）

- 防御阶段：监听 `EventBus.defense_key_pressed` → 播放对应格挡/攻击/闪避动画。
- 攻击阶段：监听 `EventBus.attack_performed` → 播放轻/重/蓄力/回复动画。
- 动画名称由 `CharacterAnimConfig` 资源提供，支持 `AnimatedSprite2D` 优先，回退到 `AnimationPlayer`。
- 防御阶段动画速率同步到节拍间隔（`beat_sync = true`）。

### 4.8 分数与血量（ScoreManager）

#### 数值配置（均通过 Inspector/Resource 配置）

| 轨道 | 判定 | boss_damage | player_health |
|------|------|------------|---------------|
| GUARD（J） | PERFECT | 5 | +10 |
| GUARD（J） | MISS | 0 | -20 |
| HIT（I） | PERFECT | 15 | +3 |
| HIT（I） | MISS | 0 | -15 |
| DODGE（L） | PERFECT | 8 | +5 |
| DODGE（L） | MISS | 0 | -10 |

（GREAT/GOOD 成比例递减，具体值见 Inspector）

默认上限：`max_player_health = 100`，`max_boss_health = 500`，`max_boss_energy = 100`。

BOSS 精力耗尽（`boss_energy_depleted`）→ 触发攻击阶段。

## 九、添加新功能指南

### 添加新信号

1. 在 `EventBus.gd` 中声明信号（带类型注解）。
2. 在 `_suppress_unused_signal_warnings()` 函数中添加 `.emit()` 占位调用以消除 UNUSED_SIGNAL 警告。
3. 在目标节点的 `_ready()` 中通过 `EventBus.<signal>.connect(callback)` 注册监听。

### 添加新铺面

1. 准备 `.sm` 文件，放入 `assets/charts/`。
2. 准备分轨音乐（main/drum/bass），放入 `assets/music/<曲名>/`。
3. 创建 `MusicConfig` 资源，配置音频路径与音量。
4. 在 `BeatManager` 的 `chart_sm_path` 中指定 `.sm` 路径，`MusicPlayer` 中指定 `MusicConfig`。

### 添加新轨道动画

1. 创建包含 `AnimatedSprite2D`（或 `AnimationPlayer`）子节点的 `.tscn` 场景。
2. 在 `TrackAnimationConfig` 资源中配置对应轨道的 `scene`、`animation_name`、`attack_end_frame`。
3. 如需预警特效，同样创建预警场景并配置 `warn_scene`。
